텍스트 레벨의 특징
섹션이나 그룹핑 컨텐츠를 보면 한 줄을 모두 차지하는 성질을 가지고 있다.
그룹핑 컨텐츠에서 제일 대표적인 것-ul, li
텍스트 레벨은 딱 컨텐츠 크기만큼만 차지한다.
예-a, span
스크린 리더를 위해 h1~6 태그 붙여주고 css로 hidden 해주면 된다

br-줄바꿈.
주석은 실무에서 정말정말 불가피할 경우에만 쓴다. 그것도 코드낭비라고 보기 때문에
포트폴리오에는 주석을 잘 넣어서 여러분의 의도를 설명해라
wbr-단어가 너무 길 때, 해당 지점에서 개행이 될 수 있도록 한다.
a href-경로 지정. href 속성을 사용하지 않고 자바스크립트로 경로를 지정할 수 있지만 웹 접근성에 위배되므로 href 속성을 사용하는게 좋다.
         a 태그 안에 a 태그를 또 넣지 않는다. 
         섹션, 그룹핑 컨텐츠 요소들은 텍스트 레벨 요소의 자식으로 사용되지 않지만, a 태그만 예외.
         페이지 안에서 이동하고 싶을 때 a href="#글자"
         a href=""download="파일명" ->다운로드 팝업 생성
         a href="tel:전화번호"->전화를 실행시킬 수 있는 어플/서비스 같은게 뜸
         a href="mailto:이메일주소"->메일 보낼 수 있는 앱이 열림
b- 굵은 글꼴 표현하고 싶을 때. 쓰지마세요.
strong- 굵은 글꼴을 보여주고 중요도 강조하고 싶을 때.
i- 기울임 글꼴. 문단에서 주 언어와 다른 언어로 표기된 부분, 등장인물의 생각이 표기되어 있는 부분 등에 쓰인다.
em-기울임 글꼴+강조
dfn-현재 문맥에서 정의하고 있는 용어를 나타낼 때. 하나 혹은 두개만 정의할 때 사용.
     문서에서 최초로 나타났을 때 사용.
abbr-줄인말, 약자를 나타내고 싶을 때. 보통은 홀로 쓰이고 dfn 태그로 감싸주기도 한다.
sub-아랫첨자 사용. 화학기호나 수학공식 등 첨자 기호를 이용해야할때만. 
sup-윗첨자 사용. 화학기호나 수학공식 등 첨자 기호를 이용해야할때만.
span-별다른 의미 없음. 줄바꿈 없이 영역을 묶는 용도로 사용.
       여러 요소를 묶어 컨트롤하기 위한 용도로 보통 id나 class를 사용하기도 한다.
       div와 마찬가지로 최후 수단으로 사용해야한다.

embedded contents - 콘텐츠를 내장하는 요소
<이미지 관련 태그>
img-이미지 삽입할 때 쓰는 태그. 
      <img src="a.jpg" alt=""> ->스크린 리더가 읽지 않는다
      <img src="a.jpg"> -> 스크린 리더가 'a'를 읽는다.
      <img src="a.jpg" alt="이미지"> ->스크린 리더가 '이미지'를 읽는다.
      srcset 속성은 브라우저 내에서 자동으로 최적의 이미지를 선택해서 출력해준다. 
      다양한 크기를 가지는 동일 이미지를 최소 2개 이상 사용할 때 사용한다.
      srcset 속성의 값으로 "x서술자"와 "w서술자"가 있다.
     일단 반드시 알아야 하는 것은 이 두 서술자는 함께 사용할 수 없으며
     srcset은 반드시 src 속성과 같이 사용해야한다. 
     srcset이 동작이 안되면 src를 내보내는 백업 개념으로!
      alt 부분에 상세한 설명이 필요하다. 
picture-소스 요소와 이미지 요소를 통해 각기 다른 디스플레이 혹은 디바이스에 따라 조건에 맞는 이미지를 보여주는 요소
          크기가 각기 다른 a, b, c jpg가 있을 때
	<picture>
            		<source srcset="./sample/a.jpg" media="(min-width:960px)" /> 
            		<source srcset="./sample/b.jpg" media="(min-width:620px)" />
            		<img src="./sample/c.jpg" alt="귀여운 아기 팽귄들" />
        	</picture>
         하면 사진 크기가 960px 미만이 되면 b.jpg으로 나오고 620px 미만으로 나오면 더 작은 c.jpg가 나온다.
         picture에서 사용하는 방식은 점진적 향상 기법이라고 한다.
         기본적으로 예전 기술 환경에서 동작할 수 있는 기능을 구현해두고, 최신 기술을 사용할 수 있는 환경에서는 최신 기술을 제공해서 더 나은 사용자 경험을 준다.
         여기서 예전 기술 환경에서 동작할 수 있는 기능이란 <picture> 태그 안의 <img>태그를 의미하고,
         최신 기술은 <source>태그를 통해 브라우저가 적합한 이미지를 결정하게 하는 것이라고 생각하면 된다.
         img태그와 picture태그 중 어떤게 효율적인지 생각하고 태그 선택해야한다.
         picture를 사용할떄는 포멧을 바꾸고 싶을때 사용하고 나머지는 img만 사용해도 무방하다.

<미디어 관련 태그>
iframe- 현재 html 페이지에서 또다른 html 페이지를 보여주고 싶을 떄. 동영상을 가져올 때도 사용.
          해커가 이용해서 중간자공격을 통해 해킹할 수 있기 때문에 보통은 막아놓는다. 
          중간자공격을 막기 위해 유저 인증, 유저 접속 지역 설정, iframe 같은 태그 지원을 막는다.
 video- 동영상 재생할 때 쓰이는 태그
          src- 파일의 위치와 파일명을 알려주는 속성
          controls-파일 재생하는데 필요한 컨트롤러를 불러오는 속성
          autoplay-자동재생
          loop-영상 반복
          poster-영상 로딩중일 때 대신해서 화면에 보여줄 이미지 설정
          metadata-비디오 파일을 미리 로딩하진 않지만 파일의 메타데이터(예를 들어 영상길이)를 미리 가져오도록 하는 것
          none-비디오 파일 미리 로딩 금지. 사용자가 페이지 로딩할 때 더 빨라질 수 있다.
          track-audio 혹은 video의 자식으로 사용할 수 있는 태그. 여러 포맷 파일 지원 가능.
                  kind 텍스트 트랙의 종류 지정. 자막, 설명 등을 지정할 수 있음
                  srclang 텍스트 트랙의 언어 지정
	     label 텍스트 트랙 제목 지정 
          자막(subtitles)과 캡션(caption)의 차이- video에서 자막은 시청자가 모르는 언어에 대한 번역제공
				          caption은 음악이나 효과, 소리를 내는 주체에 대한 설명과 같은 모든 소리 데이터를 문자로 표현하여 보여준다. 
                      				주로 청각장애를 가진 사용자에게 제공한다. 

   

<오늘 배운 css 태그>
scroll-behavior: smooth(css) - 부드럽게 화면이 스크롤되도록. 더/덜 부드럽게 이동되도록 속도를 지정할 수는 없다.
word-break(css 태그)-화면을 줄였을 때 자동 줄바꿈이 될때 단어 글자 하나하나 끊지 않고 통째로 옮기는 것
white-space(css 태그)-공백 문자를 처리하는 법을 지정한다.
